# AGENTS.md

本文件定义 AI 编码代理在 TurtleClass 仓库中的强制工作规则。

## 1. 阅读顺序

开始任何任务前，必须阅读：

1. `PROJECT_MASTER.md`
2. `HANDOFF.md`
3. 当前阶段文档
4. 已批准 RFC
5. 相关源代码和测试

## 2. 不得自行改变需求

- 不得静默改变业务规则。
- 不得用实现方便为理由修改需求。
- 发现矛盾时写入 `docs/00_Project/OPEN_QUESTIONS.md`。
- 阻塞问题必须停止相关实现并报告。
- 非阻塞问题可以采用保守、可撤销的假设，但必须记录。

## 3. 编译与测试

每次实质性修改后：

1. 配置相关目标。
2. 编译相关目标。
3. 运行相关测试。
4. 修复发现的问题。
5. 再继续后续工作。

交付前：

- 执行完整构建。
- 执行全部测试。
- 检查编译输出。
- 检查需求覆盖。
- 更新文档。

只要构建或测试失败，不得声称完成。

不得以“与本次修改无关”为理由忽略错误。若为既有错误，仍必须定位、记录影响，并在任务范围允许时修复。

禁止：

- 删除失败测试以制造通过
- 弱化断言
- 注释掉错误路径
- 吞异常
- 返回虚假成功
- 未运行测试却声称测试通过

## 4. 代码原则

- C++23
- 不使用浮点数表示积分或徽章
- 不以姓名为主键
- Core 不依赖 UI、网络或平台 API
- UI 不复制业务逻辑
- 所有业务变化通过事件
- 不在领域逻辑外直接设置最终状态
- 不自行设计密码学算法
- 错误必须显式传播
- 日志不得包含密码、密钥或真实学生隐私

## 5. 文档更新

每次可交付变更必须更新：

- `PROJECT_MASTER.md` 的工程进度
- `HANDOFF.md`
- 当前阶段文档
- 新增或解决的开放问题
- 如有重大决策，新增 RFC/ADR

## 6. 阶段限制

### Phase 1

不得开发：

- WinUI
- Cloudflare 部署
- DeepSeek 调用
- 正式公网 API
- 与 Core 无关的功能

### Phase 2

在 Core 测试未通过前，不得开始正式同步服务。

### Phase 3

在 API 与同步协议 v1 未冻结前，不得开始正式 WinUI 集成。

## 7. 完成报告

完成任务时必须报告：

- 修改文件
- 设计选择
- 实际构建命令
- 实际测试命令
- 测试结果
- 未完成内容
- 已知风险
- 文档更新
